# EtherNet/IP

EtherNet/IP support provides config-driven tag access for Allen-Bradley PLCs. You define the PLC endpoint, optional local backplane route, and the exact tags to read or write, then access those tags by local aliases from Python.

<Warning>
EtherNet/IP support is currently exposed through `instro.unstable.ethernetip` and depends on the local native package `instro-ethernetip-python`. It is not part of the stable `instro[all]` install path yet.
</Warning>

## Quickstart

Publishing EtherNet/IP data to Nominal Core with the Nominal Instrumentation Library:

<Tabs>

<Tab title="Code">

```python
import time

from instro.lib.publishers import NominalCorePublisher
from instro.unstable.ethernetip import EtherNetIP

RID = "<dataset_rid>"  # Replace with your dataset RID.

# Instantiate device with JSON config. autostart opens and begins polling.
plc = EtherNetIP(config="compactlogix.json", autostart=True)

# Add publisher to flow numeric data to Nominal Core.
plc.add_publisher(NominalCorePublisher(dataset_rid=RID))

# Allow some time for background daemon reads.
time.sleep(10)

plc.close()
```

## Current Support

EtherNet/IP support is intentionally narrow today. It is meant for explicitly configured PLC tags where the tag names and data types are already known.

| Area | Supported today |
| --- | --- |
| PLC family | Tested on Allen-Bradley CompactLogix 5332E 1769-L32E |
| Transport | EtherNet/IP explicit messaging over TCP |
| Route paths | Direct connection, or local backplane slot hops only |
| Background polling | Batched reads for all `poll: true` tags |
| Streaming to Core/Connect | Boolean and numeric scalar tag values |
| Manual tag operations | Scalar values and strings through the native session API |
| Tag discovery | Not supported |
| UDTs | Not supported in the config-driven API |
| Arrays | Not supported in the config-driven API |

<Note>
Other Allen-Bradley Logix-family PLCs may work through the same underlying protocol, but the current implementation has only been tested against the CompactLogix 5332E 1769-L32E.
</Note>

## When to Use Each Feature

Most PLCs can be brought online with a connection, a few scalar tags, and `autostart=True`. The features below map common EtherNet/IP needs to the relevant config fields.

| If you're trying to... | Reach for... |
| --- | --- |
| Stream live scalar PLC data to Nominal Core or Connect | `timing.poll_interval` + tags with `poll: true` |
| Read several polled tags efficiently | Background batched reads, enabled automatically for `poll: true` tags |
| Write setpoints or command tags safely | `write_min` / `write_max` on numeric tags |
| Convert raw numeric PLC values to engineering units | `scale` |
| Talk to a PLC CPU behind a local chassis/backplane | `connection.route_path.hops` with `type: "backplane"` |
| Read or write a string tag manually | `instro.unstable._ethernetip.EtherNetIpSession` |

## Key Concepts

### Lifecycle Pattern

The typical EtherNet/IP workflow follows this pattern:

1. **Define your PLC tags** - Create a JSON config file or build an `EtherNetIPConfig` in Python.
2. **`EtherNetIP(config)`** - Instantiate the client with your config and optional connection override.
3. **`open()`** - Establish the native EtherNet/IP session.
4. **`start()`** - Begin background polling if timing is configured.
5. **Read or Write** - Access tags by alias with type validation and optional scaling.
6. **`close()`** - Close the session and stop background polling.

<Tip>
Pass `autostart=True` to combine steps 3 and 4. This requires a `timing` section in the config.
</Tip>

### Supported Data Types

The config-driven API supports scalar PLC tag types. The `data_type` field is required for every tag so reads can validate that the PLC returned the expected kind before publishing.

| Data Type | Python Payload | Streaming to Core/Connect | Notes |
| --- | --- | --- | --- |
| `bool` | `bool` read as `0` or `1` in measurements | Yes | Boolean reads are published as numeric samples. |
| `sint` | `int` | Yes | 8-bit signed integer. |
| `int` | `int` | Yes | 16-bit signed integer. |
| `dint` | `int` | Yes | 32-bit signed integer. |
| `lint` | `int` | Yes | 64-bit signed integer. |
| `usint` | `int` | Yes | 8-bit unsigned integer. |
| `uint` | `int` | Yes | 16-bit unsigned integer. |
| `udint` | `int` | Yes | 32-bit unsigned integer. |
| `ulint` | `int` | Yes | 64-bit unsigned integer. |
| `real` | `float` | Yes | 32-bit floating point. |
| `lreal` | `float` | Yes | 64-bit floating point. |
| `string` | `str` | No | Strings can be read/written manually, but cannot be polled for streaming. |

<Warning>
UDTs and arrays are not supported by the config-driven EtherNet/IP API yet. Configure scalar tags only. If a PLC value returns as a structured value, it is not publishable through `EtherNetIP.read()` or background polling.
</Warning>

### Strings

String tags are supported for manual operations, but not for streaming measurements to Core or Connect. This is a Core/Connect measurement-streaming limitation rather than a PLC protocol limitation.

In config files, string tags must set `poll: false`:

```json
{
  "alias": "recipe_name",
  "tag_name": "RecipeName",
  "data_type": "string",
  "poll": false
}
```

Use the private native session for direct string reads or writes when you need the value in Python:

```python
from instro.unstable._ethernetip import EtherNetIpSession, PlcValue

with EtherNetIpSession("192.168.1.10:44818", route_path_slots=[0]) as session:
    current = session.read_tag("RecipeName")
    print(current.value)
    session.write_tag("RecipeName", PlcValue.string("startup"))
```

The config-driven `EtherNetIP.write()` path can also write string command tags when `poll` is disabled:

```python
plc.write("recipe_name", "startup")
```

### Background Batched Reads

When `timing` is configured and polling is started, all tags with `poll: true` are read in a single batched request per polling cycle. A per-tag read or decode failure is logged and does not discard successful values from other tags in the same batch.

Only streamable scalar tags should be polled:

- `bool`
- signed and unsigned integer types
- `real`
- `lreal`

String, UDT, and array values should not be configured for polling.

### Route Paths

Route paths are limited to local backplane slot hops:

```json
{
  "connection": {
    "host": "192.168.1.10",
    "port": 44818,
    "route_path": {
      "hops": [
        {"type": "backplane", "slot": 0}
      ]
    }
  }
}
```

The config schema allows multiple local backplane hops:

```json
{
  "route_path": {
    "hops": [
      {"type": "backplane", "slot": 2},
      {"type": "backplane", "slot": 0}
    ]
  }
}
```

However, the implementation has only been tested where a single backplane hop is in use.

<Warning>
Network hops are not supported. You cannot configure a route path that hops through Ethernet to another PLC, remote chassis, or IP address. The only accepted hop type is `backplane`, and CIP port 1 is implied by the `slot` value.
</Warning>

### Tag Discovery

Tag discovery is not supported. Every tag must be declared explicitly in the `tags` array with:

- a local `alias`
- the PLC `tag_name`
- the expected scalar `data_type`
- optional `scale`, `poll`, `write_min`, and `write_max`

If the PLC tag's actual type does not match the configured `data_type`, reads fail with a type error instead of publishing an incorrectly decoded value.

### Scaling

Linear scaling converts between raw PLC values and physical units:

```
physical = offset + (gain * raw)
```

```json
{
  "alias": "pressure",
  "tag_name": "PressureRaw",
  "data_type": "dint",
  "scale": {"type": "linear", "gain": 0.01, "offset": 0}
}
```

Scaling is applied automatically on reads and reversed on writes. Scaling is only supported for numeric tags.

### Write Limits

Use `write_min` and `write_max` to reject unsafe writes before they reach the PLC:

```json
{
  "alias": "speed_setpoint",
  "tag_name": "SpeedSetpoint",
  "data_type": "real",
  "write_min": 0.0,
  "write_max": 2500.0
}
```

```python
plc.write("speed_setpoint", 1200.0)  # OK
plc.write("speed_setpoint", 9999.0)  # Raises ValueError
```

Write limits are checked in the units passed by the caller. If `scale` is configured, those are physical units before reverse scaling is applied.

## Configuration

Every EtherNet/IP config file must include:

- **`version`** - identifies the config schema version.
- **`protocol`** - must be `"ethernetip"` so the wrong protocol config fails clearly.
- **`device`** - metadata used for channel naming.
- **`tags`** - explicit tag definitions.

### Full Configuration Example

```json
{
  "version": 1,
  "protocol": "ethernetip",
  "device": {
    "name": "line_plc",
    "description": "CompactLogix PLC for line telemetry",
    "manufacturer": "Allen-Bradley",
    "model": "CompactLogix 5332E 1769-L32E"
  },
  "connection": {
    "host": "192.168.1.10",
    "port": 44818,
    "route_path": {
      "hops": [
        {"type": "backplane", "slot": 0}
      ]
    }
  },
  "timing": {
    "poll_interval": 1.0
  },
  "tags": [
    {
      "alias": "ready",
      "tag_name": "Ready",
      "data_type": "bool"
    },
    {
      "alias": "line_speed",
      "tag_name": "LineSpeed",
      "data_type": "real",
      "write_min": 0.0,
      "write_max": 2500.0
    },
    {
      "alias": "pressure",
      "tag_name": "PressureRaw",
      "data_type": "dint",
      "scale": {"type": "linear", "gain": 0.01, "offset": 0}
    },
    {
      "alias": "recipe_name",
      "tag_name": "RecipeName",
      "data_type": "string",
      "poll": false
    }
  ]
}
```

### Device Section

Metadata about the physical PLC. The `name` field is used as a prefix for channel names when publishing data.

| Field | Required | Description |
| --- | --- | --- |
| `name` | Yes | Device name, used as channel name prefix |
| `description` | No | Human-readable description |
| `manufacturer` | No | Device manufacturer |
| `model` | No | Device model number |

### Connection Section

| Field | Required | Default | Description |
| --- | --- | --- | --- |
| `host` | Yes | - | PLC IP address or hostname |
| `port` | No | `44818` | EtherNet/IP TCP port |
| `route_path` | No | - | Optional local backplane route path |

#### Route Path

| Field | Required | Default | Description |
| --- | --- | --- | --- |
| `hops` | No | `[]` | Ordered list of local backplane hops |

Each hop must be:

| Field | Required | Description |
| --- | --- | --- |
| `type` | Yes | Must be `"backplane"` |
| `slot` | Yes | Backplane slot number, 0-255 |

### Tag Definitions

Each tag entry defines one aliasable PLC tag:

| Field | Required | Default | Description |
| --- | --- | --- | --- |
| `alias` | Yes | - | Unique local alias used in `read()` and `write()` |
| `tag_name` | Yes | - | PLC tag name |
| `description` | No | - | Optional human-readable description |
| `data_type` | Yes | - | Expected scalar PLC data type |
| `scale` | No | - | Optional linear scaling for numeric tags |
| `poll` | No | `true` | Include this tag in background polling |
| `write_min` | No | - | Minimum allowed write value for numeric tags |
| `write_max` | No | - | Maximum allowed write value for numeric tags |

### Timing Section

| Field | Required | Default | Description |
| --- | --- | --- | --- |
| `poll_interval` | Yes | - | Polling interval in seconds (0.01-10.0) |

When the `timing` section is present, all tags with `poll: true` are read at the specified interval in a background thread. Successful values are published and buffered for retrieval.

### Validation Rules

The configuration is validated at load time:

- `protocol` must be `"ethernetip"`.
- Tag aliases must be unique.
- Every tag must declare `data_type`.
- `data_type` must be one of the supported scalar types.
- `scale` is only allowed on numeric tags.
- `write_min` and `write_max` are only allowed on numeric tags.
- `write_min` must be less than or equal to `write_max`.
- Integer write limits must fit in the configured PLC integer type after scaling.
- String tags must set `poll: false`.
- Route path hops must be local backplane hops with slots from 0 to 255.
- Unsupported route hop fields such as explicit CIP ports or Ethernet addresses are rejected.

```python
from instro.protocols.common_types import DeviceInfo
from instro.unstable.ethernetip import (
    EtherNetIP,
    EtherNetIPConfig,
    EtherNetIPConnection,
    TagDef,
    TimingConfig,
)

config = EtherNetIPConfig(
    device=DeviceInfo(name="line_plc"),
    timing=TimingConfig(poll_interval=1.0),
    tags=[
        TagDef(alias="ready", tag_name="Ready", data_type="bool"),
        TagDef(alias="line_speed", tag_name="LineSpeed", data_type="real"),
    ],
)

connection = EtherNetIPConnection(
    host="192.168.1.10",
    route_path={"hops": [{"type": "backplane", "slot": 0}]},
)

plc = EtherNetIP(config, connection=connection, autostart=True)
```

## Reading and Writing

Channel names in returned `Measurement` and `Command` objects follow the pattern `{device_name}.{tag_alias}`. Write commands append `.cmd` to the channel name.

```python
measurement = plc.read("line_speed")
print(measurement.channel_data["line_plc.line_speed"])

command = plc.write("line_speed", 1200.0)
print(command.channel_data["line_plc.line_speed.cmd"])
```

## Background Polling

Use `autostart=True` to open the session and begin background polling immediately:

```python
from instro.lib.publishers import NominalCorePublisher
from instro.unstable.ethernetip import EtherNetIP

plc = EtherNetIP(config="compactlogix.json", autostart=True)
plc.add_publisher(NominalCorePublisher(dataset_rid="<dataset_rid>"))

recent_speed = plc.get_channel("line_plc.line_speed", length=100)
plc.close()
```

Or start explicitly:

```python
plc = EtherNetIP(config="compactlogix.json")
plc.open()
plc.start()
```

## Error Handling

EtherNet/IP raises descriptive errors for common issues:

| Error | Meaning |
| --- | --- |
| `RuntimeError: EtherNet/IP support requires the local native package 'instro-ethernetip-python'` | The native PyO3 package is not installed in the environment. |
| `RuntimeError: EtherNet/IP client not connected. Call open() first.` | Attempting to read or write before opening. |
| `TypeError: Tag '{alias}' expected PLC kind ...` | The PLC returned a different scalar type than the config declared. |
| `ValueError: Tag '{alias}' value ... is below write_min` | A write was rejected by local bounds checking. |
| `ValueError: Tag '{alias}' raw value ... is out of range for ...` | A raw integer write does not fit in the configured PLC integer type. |
| Native `EtherNetIpError` | The underlying EtherNet/IP request failed, for example due to connection, CIP, or tag errors. |

## Known Limitations

- EtherNet/IP support is unstable and local-development only.
- Hardware testing has only covered Allen-Bradley CompactLogix 5332E 1769-L32E.
- Route paths support local backplane hops only. Network hops are unsupported.
- Only a single backplane hop has been tested.
- Tag discovery is unsupported.
- UDTs are unsupported in the config-driven API.
- Arrays are unsupported in the config-driven API.
- String tags cannot be streamed to Core or Connect; read/write them manually instead.
