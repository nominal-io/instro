# EtherNet/IP

EtherNet/IP config files declare an Allen-Bradley PLC endpoint, an optional local backplane route, and the scalar tags Nominal reads or writes. `EtherNetIP` exposes those tags by local alias from Python and can publish boolean and numeric tag values to Nominal Core or Nominal Connect.

<Warning>
The EtherNet/IP client lives under `instro.unstable.ethernetip` and requires the local native package `instro-ethernetip-python`. It is not part of the stable `instro[all]` install path.
</Warning>

## Quickstart

This example opens a CompactLogix connection, polls scalar tags, and publishes numeric samples to Nominal Core.

<Tabs>

<Tab title="Code">

```python
import time

from instro.lib.publishers import NominalCorePublisher
from instro.unstable.ethernetip import EtherNetIP

RID = "<dataset_rid>"  # Nominal Core dataset RID.

# autostart opens the session and starts polling.
plc = EtherNetIP(config="compactlogix.json", autostart=True)

# Publish boolean and numeric tag samples to Nominal Core.
plc.add_publisher(NominalCorePublisher(dataset_rid=RID))

# Let the background daemon collect one polling window.
time.sleep(10)

plc.close()
```

## Current support

EtherNet/IP support is intentionally narrow. It targets PLC tags whose names and scalar data types are known before runtime.

| Area | Supported today |
| --- | --- |
| PLC family | Tested on Allen-Bradley CompactLogix 5332E 1769-L32E |
| Transport | EtherNet/IP explicit messaging over TCP |
| Route paths | Direct connection, or local backplane slot hops only |
| Background polling | Batched reads for all `poll: true` tags |
| Streaming to Nominal Core or Nominal Connect | Boolean and numeric scalar tag values |
| Manual tag operations | Scalar values and strings through the native session API |
| Tag discovery | Not supported |
| UDTs | Not supported in the config-driven API |
| Arrays | Not supported in the config-driven API |

<Note>
Other Allen-Bradley Logix-family PLCs may work through the same underlying protocol. Current hardware testing covers only CompactLogix 5332E 1769-L32E.
</Note>

## When to use each feature

A connection, scalar tag definitions, and `autostart=True` cover the common path. Use the fields below for specific EtherNet/IP requirements.

| Requirement | Configuration |
| --- | --- |
| Stream live scalar PLC data to Nominal Core or Nominal Connect | `timing.poll_interval` + tags with `poll: true` |
| Read all polled tags in one request | Background batched reads for `poll: true` tags |
| Reject out-of-range setpoints before PLC writes | `write_min` / `write_max` on numeric tags |
| Convert raw numeric PLC values to engineering units | `scale` |
| Reach a PLC CPU behind a local chassis/backplane | `connection.route_path.hops` with `type: "backplane"` |
| Read or write a string tag manually | `instro.unstable._ethernetip.EtherNetIpSession` |

## Key concepts

### Lifecycle

The EtherNet/IP workflow has six moves:

1. Define PLC tags in a JSON config file or an `EtherNetIPConfig`.
2. Instantiate `EtherNetIP(config)` with an optional connection override.
3. Call `open()` to establish the native EtherNet/IP session.
4. Call `start()` to begin background polling when timing is configured.
5. Read or write tags by alias with type validation and optional scaling.
6. Call `close()` to close the session and stop background polling.

<Tip>
Pass `autostart=True` to combine steps 3 and 4. This requires a `timing` section in the config.
</Tip>

### Supported data types

The config-driven API supports scalar PLC tag types. The `data_type` field is required for every tag so reads can validate that the PLC returned the expected kind before publishing.

| Data type | Python payload | Streaming to Nominal Core or Nominal Connect | Notes |
| --- | --- | --- | --- |
| `bool` | `bool` read as `0` or `1` in measurements | Yes | Boolean reads are published as numeric samples. |
| `sint` | `int` | Yes | 8-bit signed integer. |
| `int` | `int` | Yes | 16-bit signed integer. |
| `dint` | `int` | Yes | 32-bit signed integer. |
| `lint` | `int` | Yes | 64-bit signed integer. |
| `usint` | `int` | Yes | 8-bit unsigned integer. |
| `uint` | `int` | Yes | 16-bit unsigned integer. |
| `udint` | `int` | Yes | 32-bit unsigned integer. |
| `ulint` | `int` | Yes | 64-bit unsigned integer. |
| `real` | `float` | Yes | 32-bit floating point. |
| `lreal` | `float` | Yes | 64-bit floating point. |
| `string` | `str` | No | Manual reads and writes only. |

<Warning>
The config-driven EtherNet/IP API supports scalar tags only. Structured values cannot be declared as a tag `data_type`; if a configured scalar tag actually returns a structured value, it is treated as an expected-versus-actual PLC kind mismatch. A manual `EtherNetIP.read()` raises `TypeError`, while background polling logs a decode warning for that tag and continues publishing any other successful reads from the same batch.
</Warning>

### Strings

Nominal Core and Nominal Connect measurements accept numeric samples in this path, so string tags support manual operations only. The PLC protocol can still read and write string tags.

In config files, string tags must set `poll: false`:

```json
{
  "alias": "recipe_name",
  "tag_name": "RecipeName",
  "data_type": "string",
  "poll": false
}
```

Use the private native session for Python-side string reads or writes:

```python
from instro.unstable._ethernetip import EtherNetIpSession, PlcValue

with EtherNetIpSession("192.168.1.10:44818", route_path_slots=[0]) as session:
    current = session.read_tag("RecipeName")
    print(current.value)
    session.write_tag("RecipeName", PlcValue.string("startup"))
```

The config-driven `EtherNetIP.write()` path also writes string command tags when `poll` is disabled:

```python
plc.write("recipe_name", "startup")
```

### Background batched reads

With `timing` configured and polling started, `EtherNetIP` reads all tags with `poll: true` in a single batched request per polling cycle. EtherNet/IP logs a read or decode failure on one tag without discarding successful values from other tags in the same batch.

Poll only streamable scalar tags:

- `bool`
- signed and unsigned integer types
- `real`
- `lreal`

Keep string, UDT, and array values out of background polling.

### Route paths

Route paths are limited to local backplane slot hops:

```json
{
  "connection": {
    "host": "192.168.1.10",
    "port": 44818,
    "route_path": {
      "hops": [
        {"type": "backplane", "slot": 0}
      ]
    }
  }
}
```

The config schema allows multiple local backplane hops:

```json
{
  "route_path": {
    "hops": [
      {"type": "backplane", "slot": 2},
      {"type": "backplane", "slot": 0}
    ]
  }
}
```

Current testing has covered one backplane hop.

<Warning>
Network hops are not supported. A route path cannot hop through Ethernet to another PLC, remote chassis, or IP address. The only accepted hop type is `backplane`, and CIP port 1 is implied by the `slot` value.
</Warning>

### Tag discovery

Tag discovery is not supported. Every tag must be declared explicitly in the `tags` array with:

- a local `alias`
- the PLC `tag_name`
- the expected scalar `data_type`
- optional `scale`, `poll`, `write_min`, and `write_max`

If the PLC tag's actual type differs from the configured `data_type`, reads fail with a type error instead of publishing an incorrectly decoded value.

### Scaling

Linear scaling converts between raw PLC values and physical units:

```
physical = offset + (gain * raw)
```

```json
{
  "alias": "pressure",
  "tag_name": "PressureRaw",
  "data_type": "dint",
  "scale": {"type": "linear", "gain": 0.01, "offset": 0}
}
```

EtherNet/IP applies scaling on reads and reverses it on writes. Scaling supports numeric tags only.

### Write limits

Set `write_min` and `write_max` to reject unsafe writes before they reach the PLC:

```json
{
  "alias": "speed_setpoint",
  "tag_name": "SpeedSetpoint",
  "data_type": "real",
  "write_min": 0.0,
  "write_max": 2500.0
}
```

```python
plc.write("speed_setpoint", 1200.0)  # OK
plc.write("speed_setpoint", 9999.0)  # Raises ValueError
```

EtherNet/IP checks write limits in the units passed by the caller. With `scale` configured, those units are physical units before reverse scaling.

## Configuration

Every EtherNet/IP config file must include:

- **`version`**: identifies the config schema version.
- **`protocol`**: must be `"ethernetip"` so the wrong protocol config fails clearly.
- **`device`**: metadata used for channel naming.
- **`tags`**: explicit tag definitions.

### Full configuration example

```json
{
  "version": 1,
  "protocol": "ethernetip",
  "device": {
    "name": "line_plc",
    "description": "CompactLogix PLC for line telemetry",
    "manufacturer": "Allen-Bradley",
    "model": "CompactLogix 5332E 1769-L32E"
  },
  "connection": {
    "host": "192.168.1.10",
    "port": 44818,
    "route_path": {
      "hops": [
        {"type": "backplane", "slot": 0}
      ]
    }
  },
  "timing": {
    "poll_interval": 1.0
  },
  "tags": [
    {
      "alias": "ready",
      "tag_name": "Ready",
      "data_type": "bool"
    },
    {
      "alias": "line_speed",
      "tag_name": "LineSpeed",
      "data_type": "real",
      "write_min": 0.0,
      "write_max": 2500.0
    },
    {
      "alias": "pressure",
      "tag_name": "PressureRaw",
      "data_type": "dint",
      "scale": {"type": "linear", "gain": 0.01, "offset": 0}
    },
    {
      "alias": "recipe_name",
      "tag_name": "RecipeName",
      "data_type": "string",
      "poll": false
    }
  ]
}
```

### Device section

The `device` section describes the physical PLC. EtherNet/IP uses the `name` field as a prefix for published channel names.

| Field | Required | Description |
| --- | --- | --- |
| `name` | Yes | Device name, used as channel name prefix |
| `description` | No | Human-readable description |
| `manufacturer` | No | Device manufacturer |
| `model` | No | Device model number |

### Connection section

| Field | Required | Default | Description |
| --- | --- | --- | --- |
| `host` | Yes | - | PLC IP address or hostname |
| `port` | No | `44818` | EtherNet/IP TCP port |
| `route_path` | No | - | Optional local backplane route path |

#### Route path

| Field | Required | Default | Description |
| --- | --- | --- | --- |
| `hops` | No | `[]` | Ordered list of local backplane hops |

Each hop must be:

| Field | Required | Description |
| --- | --- | --- |
| `type` | Yes | Must be `"backplane"` |
| `slot` | Yes | Backplane slot number, 0-255 |

### Tag definitions

Each tag entry defines one aliasable PLC tag:

| Field | Required | Default | Description |
| --- | --- | --- | --- |
| `alias` | Yes | - | Unique local alias used in `read()` and `write()` |
| `tag_name` | Yes | - | PLC tag name |
| `description` | No | - | Optional human-readable description |
| `data_type` | Yes | - | Expected scalar PLC data type |
| `scale` | No | - | Optional linear scaling for numeric tags |
| `poll` | No | `true` | Include this tag in background polling |
| `write_min` | No | - | Minimum allowed write value for numeric tags |
| `write_max` | No | - | Maximum allowed write value for numeric tags |

### Timing section

| Field | Required | Default | Description |
| --- | --- | --- | --- |
| `poll_interval` | Yes | - | Polling interval in seconds (0.01-10.0) |

When the `timing` section is present, `EtherNetIP` reads all tags with `poll: true` at the specified interval in a background thread. Successful values are published and buffered for retrieval.

### Validation rules

`EtherNetIP` validates configuration at load time:

- `protocol` must be `"ethernetip"`.
- Tag aliases must be unique.
- Every tag must declare `data_type`.
- `data_type` must be one of the supported scalar types.
- Only numeric tags can use `scale`.
- Only numeric tags can use `write_min` and `write_max`.
- `write_min` must be less than or equal to `write_max`.
- Integer write limits must fit in the configured PLC integer type after scaling.
- String tags must set `poll: false`.
- Route path hops must be local backplane hops with slots from 0 to 255.
- Configuration validation rejects unsupported route hop fields such as explicit CIP ports or Ethernet addresses.

```python
from instro.protocols.common_types import DeviceInfo
from instro.unstable.ethernetip import (
    EtherNetIP,
    EtherNetIPConfig,
    EtherNetIPConnection,
    TagDef,
    TimingConfig,
)

config = EtherNetIPConfig(
    device=DeviceInfo(name="line_plc"),
    timing=TimingConfig(poll_interval=1.0),
    tags=[
        TagDef(alias="ready", tag_name="Ready", data_type="bool"),
        TagDef(alias="line_speed", tag_name="LineSpeed", data_type="real"),
    ],
)

connection = EtherNetIPConnection(
    host="192.168.1.10",
    route_path={"hops": [{"type": "backplane", "slot": 0}]},
)

plc = EtherNetIP(config, connection=connection, autostart=True)
```

## Read and write

Returned `Measurement` and `Command` objects use channel names in the pattern `{device_name}.{tag_alias}`. Write commands append `.cmd` to the channel name.

```python
measurement = plc.read("line_speed")
print(measurement.channel_data["line_plc.line_speed"])

command = plc.write("line_speed", 1200.0)
print(command.channel_data["line_plc.line_speed.cmd"])
```

## Background polling

Use `autostart=True` to open the session and begin background polling immediately:

```python
from instro.lib.publishers import NominalCorePublisher
from instro.unstable.ethernetip import EtherNetIP

plc = EtherNetIP(config="compactlogix.json", autostart=True)
plc.add_publisher(NominalCorePublisher(dataset_rid="<dataset_rid>"))

recent_speed = plc.get_channel("line_plc.line_speed", length=100)
plc.close()
```

Start explicitly when the session should open before polling begins:

```python
plc = EtherNetIP(config="compactlogix.json")
plc.open()
plc.start()
```

## Error handling

EtherNet/IP raises these errors for common failures:

| Error | Meaning |
| --- | --- |
| `RuntimeError: EtherNet/IP support requires the local native package 'instro-ethernetip-python'` | The native PyO3 package is not installed in the environment. |
| `RuntimeError: EtherNet/IP client not connected. Call open() first.` | A read or write occurred before `open()`. |
| `TypeError: Tag '{alias}' expected PLC kind ...` | The PLC returned a different kind than the configured scalar `data_type`, including a structured value for a scalar tag. |
| `ValueError: Tag '{alias}' value ... is below write_min` | A write was rejected by local bounds checking. |
| `ValueError: Tag '{alias}' raw value ... is out of range for ...` | A raw integer write does not fit in the configured PLC integer type. |
| Native `EtherNetIpError` | The underlying EtherNet/IP request failed because of a connection, CIP, or tag error. |

## Known limitations

- EtherNet/IP support is unstable and local development only.
- Hardware testing covers Allen-Bradley CompactLogix 5332E 1769-L32E.
- Route paths support local backplane hops only. Network hops are unsupported.
- Current testing covers one local backplane hop.
- Tag discovery is unsupported.
- UDTs are unsupported in the config-driven API.
- Arrays are unsupported in the config-driven API.
- String tags cannot be streamed to Nominal Core or Nominal Connect; read or write them manually.
